# Reflection Nathan

### How I wrote prompts

I wrote prompts by gathering all the context I needed for to be able to make the propmpts as effective as possible for what task I am trying to accomplish at that givin time. Once done with this step, I move on to make a step by step prompt on how it should approach and accomplish the goal I want to make. This gives AI better clarity on how it should approach the result and get the best and most accurate possible results. I tried to be as detailed as possible so the AI bot has a less chance of messing up and giving incorrect solutions.

### How I validated AI output

My best approach of seeing if AI output was validated was checking to make sure all the created files and folders were in the right place and the correct ones produced. I also went back to my prompts to make sure that everything I wanted to produce in that prompt to be in there so AI can give me the right results.


### What I learned about spec-driven development

I learned so many things when it came to spec-driven development in this project. The first thing I learned about spec-driven development was that implementation can be sped up a lot faster and get great results because it makes me have to use my brain to define the data and scope of what results I am looking for. I also had create an order of logical arrangements for this prompts so I can get the correct data and results to be able to build on from that point and just keeping repeating the cycle over and over until I got my finished product. My favorite part was being able to use my creativity and build out the fomula first to create ordered prompts instead of just trying to create everything at one time which is very ineffective.

## What they would change next 

In the future, I would be focused more on is making my prompts a little more in the logical flow because some prompts I created had some redundant or illogical data that was sprinkled in there that made the AI produced some inaccurate results. I also need to be more aware on how I word thing in my propmt because I had issues where I was to vague on certain things and that cause the results to be a little vague.
