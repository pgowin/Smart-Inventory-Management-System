# Reflection 2
How I wrote Prompts
Writing prompts was initially very simple. However, we learned how to be much more specific and detailed in our prompts. This improved the accuracy significanty as to what the LLM actually did.
How they validated AI Output
We vailidated AI output in a few different ways. The first being reading the code line by line which we ultimately decided was not the most effecient. So we began looking at the dev server and verifying what was happening in the code with what we could see. 
What they learned about spec-driven development
I learned that spec-driven development is significantly faster than normal development. It is also completely different than straight up vibe coding. With vibe coding anyone can do it, and it is not very thought out. Spec-driven development is much more thought out and planned as opposed to just typing in basic sentence prompts like vibe coding. 
What would I change next time?
Next time I would want to focus more on fixing some of the UI earlier on. This would remove several of the later prompts to reduce the amount of correction later. If we were much clearer in the start of the project it woudl've removed these earlier fixes. It also reaquires a lot of learning how to write these prompts in better language. Learning specific vocabulary to be able to accurately describe the features that you want as well as how they look from a GUI perspective.
